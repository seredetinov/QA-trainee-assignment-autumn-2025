## Инструкция

Для просмотра выполненного задания 1 откройте файл **BUGS.md** в директории **task1**.

Для просмотра тест-кейсов и баг-репортов по заданию 2.1 откройте файлы **TESTCASES.md** и **BUGS.md**, соответственно, в директории **task2_1**.

При реализации автотестов API использовался язык **Java** в связке с фреймворками **REST Assured** и **JUnit**.

Для взаимодействия непосредственно с проектом автотестов API необходимо выполнить ряд действий.

1. Склонируйте к себе этот репозиторий или скачайте zip архив по [ссылке](https://github.com/seredetinov/QA-trainee-assignment-autumn-2025/archive/refs/heads/main.zip) и распакуйте его.
2. Проект автотестов API - это папка **task2_1**. Откройте её в удобной для вас IDE.
3. В проекте используется JDK 16, поэтому убедитесь, что у вас установлена не более ранняя версия.
4. Откройте файл **pom.xml** и выполните синхронизацию зависимостей проекта, чтобы IDE выполнила загрузку необходимых библиотек, если она не произошла автоматически.

Теперь Вы можете взаимодействовать с автотестами API.

Непосредственно запустить тесты Вы можете следующими способами:

- Запустить тесты каждой ручки отдельно - для этого используйте тестовые классы в пакете **src/test/java/tests**
  
ИЛИ

- Запустить сразу весь тестовый набор - для этого используйте файл **TestsRunner.java** в пакете **src/test/java/**

Каждый автотест содержит аннотацию **@DisplayName** в которой указан номер, соответствующий тест-кейсу из файла **TESTCASES.md**
